# 共通アセットの洗い出しと流用手順

次期プロジェクトである「ボードゲームのVRChat化」において、既存の「Metafes2025」プロジェクトから流用（再利用）可能な共通アセットのリストと、その流用・移行手順をまとめます。

## 1. 流用可能な共通コンポーネント（UdonSharp）
既存のシューティングのロジック（GunやEnemyなど）に依存しておらず、どんなワールドでも利用可能な汎用スクリプトです。

*   **`VRCDebugUI.cs` (デバッグ用UI)**
    *   **用途**: ロジックが複雑になるボードゲーム開発において、ワールド内で変数の値や進行状況をログとして視覚的に確認するために必須です。
    *   **特徴**: 常にプレイヤーの目の前に追従し、リングバッファでログが流れる優秀な作りになっています。
*   **`AudioTrigger.cs` (サウンド制御)**
    *   **用途**: 特定のエリア（コライダー）に入った時にBGMを再生/停止したり、ボタン操作に伴う効果音を鳴らしたりするのに使えます。
*   **`SkyboxSwitcher.cs` (背景切り替え)**
    *   **用途**: ボードゲームにおける「フェーズの移行（例：昼から夜への変化、クライマックス演出）」で、空間全体の雰囲気を変えるためにそのまま使えます。
*   **`ResetButton.cs` / `TeleportToStart.cs` (リセット・復帰系)**
    *   **用途**: プレイヤーがワールドの外に落ちてしまった場合の復帰ギミックや、ゲーム盤の初期化トリガーとして応用可能です。

## 2. 他プロジェクトへの流用（エクスポート）方法
これらのスクリプトや、それらがアタッチされたPrefabを新しいプロジェクトへ持ち込む推奨手順は**「Unity Package ( .unitypackage )」**を利用する方法です。

### 手順（Metafes2025プロジェクト側）
1.  Unityの **Projectウィンドウ** を開く。
2.  流用したいファイル（`VRCDebugUI.cs` やそのPrefab、`AudioTrigger.cs`など）を選択する。（`Ctrl`キーを押しながら複数選択可能）
3.  選択した状態で右クリックし、メニューから **「Export Package...」** を選択する。
4.  「Exporting package」ウィンドウが開くので、右下の **「Export」** ボタンを押し、分かりやすい名前（例: `Metafes_Common_v1.unitypackage`）をつけて適当なフォルダ（デスクトップ等）に保存する。
    *   *※このとき、必要な依存関係（関連するマテリアルなど）には自動でチェックが入りますが、VRChat SDK側のファイル（VRC.SDK3など）が含まれてしまっている場合はチェックを外してください。*

### 手順（新規ボードゲームプロジェクト側）
1.  VRChat Creator Companion (VCC) から、**新しいUdonSharp対応プロジェクト**（World）を作成し、Unityで開く。
2.  上部メニューの **Assets > Import Package > Custom Package...** を選択する。
3.  先ほど保存した `Metafes_Common_v1.unitypackage` を選択して開く。
4.  右下の **「Import」** をクリックすると、元のプロジェクトと同じフォルダ階層（例: `Assets/Projects/Scripts/`）にスクリプトやPrefabが展開され、そのまま利用できるようになります！

---
**💡 補足：Prefab化の推奨**
`VRCDebugUI` などは、スクリプト単体ではなく「TextなどのUI部品がすべて設定されたGameObject」ごとあらかじめPrefab化（`Assets/Projects/Prefabs/` にドラッグ＆ドロップして保存）しておき、その**Prefabを右クリックしてExportする**のが最も確実で簡単な流用方法です。
